# Timeline del hackathon

> Plan de 24-48h. Ajustar horas al evento real. La meta de cada bloque es **terminar algo demostrable**.

## Principio
Al final de **cada bloque**, algo funciona de punta a punta — aunque sea mínimo. Nunca dejar todo "casi listo".

## Bloques (plantilla 48h)

| Bloque | Objetivo | Entregable demostrable |
|---|---|---|
| **0 · Kickoff (h0-2)** | Alinear visión, roles, congelar MVP | Repo clonado, roles asignados, MVP acordado |
| **1 · Esqueleto (h2-8)** | Modelo canónico + seed + pantalla vacía | Un lead se carga y se ve en pantalla |
| **2 · Cerebro (h8-18)** | Subsidios + scoring deterministas | Un lead recibe score con razón |
| **3 · Conversación (h18-28)** | Perfilador que pregunta huecos | Flujo lead → perfil → score sin intervención |
| **4 · Integración (h28-36)** | Todo conectado end-to-end + recomendación | Demo completa corre sola |
| **5 · Pulido (h36-42)** | Nice-to-have de alto impacto + GIF | README con GIF, panel del asesor |
| **6 · Ensayo (h42-46)** | Ensayar pitch + plan B | Demo cronometrada <5 min, video de respaldo |
| **7 · Buffer (h46-48)** | Colchón para imprevistos | — |

## Checkpoints de decisión
- **Fin de bloque 2:** ¿el cerebro funciona? Si no, recortar alcance.
- **Fin de bloque 4:** ¿la demo corre sola? Si no, **freeze de features**, solo estabilizar.
- **Fin de bloque 5:** nada nuevo después de aquí. Solo ensayo y pulido.

## Regla de sueño
Alguien descansa mientras otro avanza. Un equipo sin dormir presenta mal. El pitch se gana descansado.
