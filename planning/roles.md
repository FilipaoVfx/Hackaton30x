# Roles

## Equipo

| Rol | Responsable | Foco | Backup |
|---|---|---|---|
| **Product / Pitch** | _(por asignar)_ | Visión, `docs/`, pitch, demo-script, hablar con el jurado | |
| **Backend / Cerebro** | _(por asignar)_ | Modelo canónico, scoring, subsidios, ruteo | |

| **Frontend / UX** | _(por asignar)_ | Pantallas, panel del asesor, que la demo se recorra sola | |
| **Datos / Automatización** | _(por asignar)_ | `datasets/`, seeders, `automation/`, integración de canal | |

> En equipos pequeños una persona cubre varios roles. Lo importante: **cada área tiene un dueño**.

## Acuerdos de trabajo
- **Rama por feature**, PRs pequeños, `main` siempre corre.
- Sincronización rápida cada bloque (mañana / tarde / noche).
- El dueño de **Pitch** protege el alcance: dice "eso es nice-to-have" cuando toca.
- Nadie hace push directo a `main` sin que la demo siga corriendo.
